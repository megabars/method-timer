# Method Timer — IntelliJ IDEA Plugin

Плагин для IntelliJ IDEA, который автоматически замеряет время выполнения Java-методов и отображает результаты прямо в редакторе над каждым методом.

![IntelliJ IDEA](https://img.shields.io/badge/IntelliJ%20IDEA-2024.3+-blue)
![Java](https://img.shields.io/badge/Java-8+-orange)
![Kotlin](https://img.shields.io/badge/Kotlin-1.9-purple)

## Как это работает

При запуске Java-приложения из IDE плагин автоматически подключает Java Agent, который замеряет время выполнения каждого метода вашего кода. Результаты отображаются как Code Vision линзы над методами:

```
⏱ 3.01 s
public void processData() {
    ...
}

⏱ 142 μs
private String formatOutput(String input) {
    ...
}
```

## Возможности

- **Автоматический замер** — не нужно добавлять аннотации или менять код
- **Только ваш код** — библиотеки и фреймворки не инструментируются
- **Реальное время** — тайминги обновляются каждые 3 секунды пока приложение работает
- **Консольные и серверные приложения** — работает и с короткими скриптами, и с Spring Boot
- **Персистентность** — результаты сохраняются между перезапусками IDE
- **Кликабельные линзы** — клик по времени показывает детали (FQN метода, точное время, дата запуска)

## Установка

### Из файла

1. Скачайте ZIP из [Releases](releases) или соберите самостоятельно
2. IntelliJ IDEA → **Settings** → **Plugins** → ⚙️ → **Install Plugin from Disk...**
3. Выберите `plugin-<version>.zip`
4. Перезапустите IDE

### Сборка из исходников

```bash
git clone <repo-url>
cd intellijplugin
./gradlew :plugin:buildPlugin
```

ZIP-дистрибутив: `plugin/build/distributions/plugin-<version>.zip`

## Использование

1. Откройте Java-проект в IntelliJ IDEA
2. Запустите приложение или тест обычным способом (Run/Debug)
3. После выполнения над методами появятся линзы с временем выполнения
4. Для долгоживущих приложений (Spring Boot) тайминги обновляются в реальном время

## Формат отображения времени

| Диапазон | Формат | Пример |
|----------|--------|--------|
| < 1 мкс | наносекунды | `750 ns` |
| < 1 мс | микросекунды | `142 μs` |
| < 1 с | миллисекунды | `3.2 ms` |
| ≥ 1 с | секунды | `1.50 s` |

## Архитектура

Плагин состоит из двух модулей:

### agent/ — Java Agent

- Подключается к JVM через `-javaagent`
- Использует [ByteBuddy](https://bytebuddy.net/) для инструментации байткода
- Инструментирует только код из директорий (не из JAR-файлов), что автоматически исключает все библиотеки
- Записывает результаты в JSONL-файл каждые 2 секунды

### plugin/ — IntelliJ Plugin

- `TimingJavaProgramPatcher` — автоматически добавляет `-javaagent` к запускам
- `TimingProcessListener` — читает результаты через polling (каждые 3 сек)
- `MethodTimingCodeVisionProvider` — отображает Code Vision линзы
- `MethodTimingStorage` — персистентное хранение данных между сессиями

## Ограничения

- Работает только при запуске через IntelliJ IDEA (Run/Debug), не при запуске через Gradle-делегирование
- Инструментируются только Java-методы (не Kotlin, не конструкторы)
- При принудительном завершении процесса (SIGKILL) последние данные могут быть потеряны

## Разработка

```bash
./gradlew :plugin:runIde          # Запустить IDE с плагином в sandbox
./gradlew :plugin:buildPlugin     # Собрать дистрибутив
./gradlew :agent:shadowJar        # Собрать только агент
```

## Технологии

- **IntelliJ Platform SDK** 2024.3+
- **ByteBuddy** 1.15.11 — инструментация байткода
- **Kotlin** 1.9.25 — код плагина
- **Java** 8 — код агента (для максимальной совместимости)
- **Gradle** 8.12 + Shadow Plugin — сборка
